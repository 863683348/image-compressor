# 图片压缩网站 · 需求文档与技术方案

> 版本：v1.0 ｜ 日期：2026-07-15 ｜ 技术路线：纯前端、隐私优先（浏览器端压缩）

---

## 1. 项目概述

### 1.1 一句话定位
一个**纯前端**的在线图片压缩工具：图片全程在用户浏览器内处理，**不上传服务器**，打开即用、免费、无水印。

### 1.2 核心价值
| 价值点 | 说明 |
|--------|------|
| 🔒 隐私优先 | 图片从不上传，断网也能用，适合处理含隐私/敏感信息的图片 |
| ⚡ 快速 | 本地处理，无网络往返，批量也快 |
| 🆓 免费无水印 | 不加 logo、不强制注册、不限次数 |
| 📦 零服务器成本 | 纯静态托管（可部署到任意静态站 / CDN / Pages） |

### 1.3 项目目标
- MVP 上线：覆盖 JPG/PNG/WebP 压缩 + 对比 + 下载，单页可用
- 增强版：支持 AVIF、格式互转、目标大小压缩、批量队列

---

## 2. 目标用户与场景

| 用户 | 场景 |
|------|------|
| 自媒体 / 博主 | 压缩配图，控制公众号/博客体积，提升加载速度 |
| 前端 / 网页开发者 | 优化网页图片资源（WebP/AVIF），降低 LCP |
| 电商运营 | 批量压缩商品主图、详情图 |
| 普通办公用户 | 压缩图片以邮件附件发送、满足上传体积限制 |
| 隐私敏感用户 | 处理含身份证/合同等敏感信息的图片，要求不上传 |

## 3. 功能需求

### 3.1 核心功能（MVP · P0）

**F1 图片上传**
- 支持拖拽上传 + 点击选择
- 支持多选、批量添加（一次可选多张，可继续追加）
- 支持粘贴（Ctrl+V）上传截图
- 支持格式：JPG / JPEG / PNG / WebP（基础解码由浏览器原生支持）
- 单文件上限：建议 50MB（超大图给提示）
- 拖拽进入时整页高亮拖放区反馈

**F2 压缩处理**
- 质量滑块（0–100%），默认 75%
- 实时显示：压缩后体积、压缩率（如 1.2MB → 280KB，↓76%）
- 可选输出格式（JPG / PNG / WebP）
- 处理在 Web Worker 中进行，不阻塞 UI

**F3 预览对比**
- 原图 vs 压缩图并排预览
- 显示各自尺寸（像素）与体积
- 滑块式前后对比（拖动分割线，左原图右压缩图）—— 加分项
- 可放大查看细节差异

**F4 下载导出**
- 单张下载（保留原文件名，加后缀或同名覆盖可选）
- 打包下载（ZIP，用 JSZip 生成）
- 复制压缩后的图片到剪贴板

### 3.2 增强功能（P1）

**F5 格式互转**
- PNG ↔ JPG ↔ WebP ↔ AVIF 互转
- JPG/PNG 转 WebP/AVIF 可大幅减小体积
- 透明通道处理：PNG 转 JPG 时可选背景色填充

**F6 目标大小压缩**
- 输入目标体积（如「压到 500KB 以内」）
- 自动二分查找最优质量参数，逼近目标但不超标

**F7 批量队列**
- 文件列表展示：缩略图、原大小、压缩后大小、状态
- 队列逐个处理，整体进度条
- 支持移除单个、清空全部
- 统计汇总：原始总大小 / 压缩后总大小 / 总节省

**F8 尺寸调整 / 裁剪**
- 按比例缩放（如 50%、或指定最长边像素）
- 等比 / 自定义宽高
- 简单裁剪（可选）

### 3.3 体验功能（P2）
- 暗色模式
- 移动端适配（触屏拖拽、响应式布局）
- 压缩参数记忆（localStorage 存储上次设置）
- File System Access API：可直接读取/保存到本地目录（支持的浏览器）
- 国际化（中/英）
- 拖拽调整文件顺序

## 5. 技术选型与架构

### 5.1 架构总览
```
┌──────────────────────── 浏览器（纯前端）────────────────────────┐
│                                                                │
│   UI 层（HTML/CSS/JS）                                         │
│     ├─ 拖放上传区    ├─ 质量滑块    ├─ 对比预览    ├─ 下载      │
│                                                                │
│   处理层（Web Worker，不阻塞主线程）                            │
│     ├─ 解码：Image / createImageBitmap（原生）                 │
│     ├─ 编码：Canvas.toBlob（JPG/PNG/WebP 原生）                │
│     ├─ 编码：WebAssembly（AVIF / 高级 WebP 编码器）            │
│     └─ 目标大小：二分查找质量参数                               │
│                                                                │
│   工具层：JSZip（打包）· File System Access API（可选）        │
│                                                                │
└────────────────────────────────────────────────────────────────┘
        ✗ 无服务器  ✗ 无上传  ✗ 无后端依赖
```

### 5.2 核心技术
- **Canvas API + `toBlob()`**：浏览器原生支持 JPG/PNG/WebP 编解码，零依赖
- **Web Workers**：压缩在独立线程，主线程保持流畅
- **WebAssembly 编解码库**：用于 AVIF 编码（浏览器原生不支持编码），按需懒加载
- **JSZip**：客户端生成 ZIP 包
- **File System Access API**：现代浏览器直接读写本地文件（降级为普通下载）

### 5.3 候选库
| 用途 | 候选 | 说明 |
|------|------|------|
| 通用压缩 | `browser-image-compression` | 成熟、API 简单，适合 MVP 快速起步 |
| AVIF/WebP 高级编码 | `@jsquash/*`（avif/webp/jpeg/png） | 基于 WASM 的 Squoosh 编解码器，质量高 |
| 打包下载 | `jszip` | 客户端 ZIP 生成标准方案 |
| 框架 | 原生 JS / Vue / React 任选 | MVP 可纯原生，无构建依赖 |

### 5.4 格式支持矩阵
| 格式 | 解码（输入） | 编码（输出） | 说明 |
|------|:---:|:---:|------|
| JPG/JPEG | ✅ 原生 | ✅ 原生 | 有损，最通用 |
| PNG | ✅ 原生 | ✅ 原生 | 无损，支持透明 |
| WebP | ✅ 原生 | ✅ 原生 | 有损/无损，体积小 |
| AVIF | ⚠️ 部分原生 | ❌ 需 WASM | 压缩率最高，编码依赖 WASM |
| GIF | ✅ 原生（单帧） | ❌ | 仅首帧或转 MP4（后续） |

## 7. 关键技术点

### 7.1 压缩实现（Canvas 原生路线）
```js
// Worker 内：质量压缩
const bitmap = await createImageBitmap(file);
const canvas = new OffscreenCanvas(bitmap.width, bitmap.height);
const ctx = canvas.getContext('2d');
ctx.drawImage(bitmap, 0, 0);
const blob = await canvas.convertToBlob({ type: 'image/webp', quality: 0.75 });
// blob 即压缩结果，对比 file.size 得压缩率
```

### 7.2 目标大小压缩（二分查找）
```
在 quality ∈ [0.1, 1.0] 区间二分：
  mid = (lo + hi) / 2
  blob = encode(mid)
  if blob.size <= target:  lo = mid   （尝试更高质量）
  else:                     hi = mid   （降低质量）
直到区间足够小，返回最优 blob
```

### 7.3 隐私保障
- 全程无 `fetch` 上传、无第三方图床请求
- 可在页面显著位置声明「图片仅在本地处理，不会上传」
- 支持断网验证（拔网线仍可压缩）

## 9. 风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| AVIF WASM 体积大 | 首屏变慢 | 按需懒加载，仅在用户选 AVIF 时加载 |
| Safari 兼容性差异 | 部分功能不可用 | 特性检测，不支持时降级 WebP/JPG |
| 超大图内存占用 | 浏览器卡顿/OOM | 限制单图尺寸，大图分块或提示 |
| `OffscreenCanvas` 兼容 | Worker 内绘图受限 | 降级为主线程 Canvas 处理 |
| 透明 PNG 转 JPG | 透明变黑 | 提供背景色填充选项（默认白） |
